### Integration Findings

#### Finding 1 — WithSource CSV Source Gap (PARTIAL, non-blocking)

**Affects:** POLISH-05, DB-03

`WithSource<T>` in `frontend/src/lib/api.ts` defines `source: "live" | "demo"`. The backend's `_load_rows_with_fallback` can return rows tagged `source: "csv"` at Layer 3. The frontend source detection:

```typescript
const source: "live" | "demo" = payload[0]?.source === "demo" ? "demo" : "live";
```

…classifies `"csv"` as `"live"`, so `isMockData` remains `false` and `DemoModeBadge` is not shown when serving CSV-tier data.

**Risk:** Low. Layer 3 activates only if **both** `smartmatch.db` and `demo.db` fail simultaneously — effectively impossible in a healthy demo environment. No crash, no error state.

**Fix (if desired):**
```typescript
source: "live" | "demo" | "csv";
const isMock = (payload[0]?.source === "demo" || payload[0]?.source === "csv");
```

#### Finding 2 — ScrollToTop scope (informational)

**Affects:** POLISH-03

`ScrollToTop` is inside `Layout` which wraps coordinator routes only. `/` and `/login` bypass Layout. Both are single-viewport pages — no scrollable content exists on them. No UX impact.
