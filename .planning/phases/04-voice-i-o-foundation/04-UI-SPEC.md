# Phase 4 — UI Design Contract: Voice I/O Foundation

> Visual and interaction contract for the Command Center voice panel.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none — custom CSS via `inject_custom_css()` | `src/ui/styles.py` |
| Preset | not applicable | Streamlit project, no shadcn |
| Component library | Streamlit native widgets + custom CSS | `src/ui/styles.py` |
| Icon library | none declared — use Streamlit emoji literals for status indicators | default |
| Font | Inter (body, 400/600) + Inter Tight (headings, 600) via Google Fonts | `src/ui/styles.py` line 28 |

**Note:** This is a Streamlit/Python project. No React component library applies. All styling

stylesheet.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding inside chat bubble meta row |
| sm | 8px | Spacing between input field and mic button; padding inside intent badge; chat bubble vertical padding |
| md | 16px | Default element spacing; chat bubble horizontal padding; card padding |
| lg | 24px | Section gap between voice panel header and chat history |
| xl | 32px | Top padding of Command Center tab content area |
| 2xl | 48px | Not used in Phase 4 |
| 3xl | 64px | Not used in Phase 4 |

Exceptions:
- Push-to-talk mic button: minimum touch target 44px × 44px (accessibility — WCAG 2.5.5 AAA)
- Chat bubble max-width: 72% of container width (prevents full-width stretch on wide displays)

## Color

All values inherited from the Academic Curator Design System in `src/ui/styles.py`. No new
colors are introduced in Phase 4.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f7f9fc` (`--surface`) | Command Center tab background, chat history scroll area |
| Secondary (30%) | `#ffffff` (`--surface-container-lowest`) | Chat bubble background (both coordinator and Jarvis), input field background |
| Accent (10%) | `#005394` (`--primary`) | Mic button active state border, "Send" button background, selected tab underline |
| Destructive | `#DC2626` (`--danger`) | Mic permission denied error banner |

Accent (`#005394`) reserved for:
1. Mic button border when recording is active (pulsing ring animation)
2. "Send" / submit button background
3. Active tab underline on "Command Center" tab
4. Left-border accent on Jarvis response bubble

Semantic colors (not part of 60/30/10 but present in existing design system):
- `#059669` (`--success`): microphone connected/ready indicator dot
- `#F59E0B` (`--intent-badge-color`): intent badge background (`[Intent: echo]` pill) — named separately to avoid collision with `--primary` accent
- `#545f72` (`--secondary`): timestamp and meta text in chat bubbles

Chat bubble color differentiation (within secondary `#ffffff` base):
- Coordinator messages: white bubble (`#ffffff`) with `--outline-variant` (`#c1c7d2`) 1px border, left-aligned
- Jarvis responses: white bubble (`#ffffff`) with `--primary` (`#005394`) 3px left border, right-aligned

## Interaction States

### Push-to-Talk Mic Button

| State | Visual | Copy |
|-------|--------|------|
| Idle | Gray mic icon, no border | "Hold to speak" (tooltip) |
| Recording | Primary blue pulsing ring (`mic-pulse` animation), red dot indicator | "Recording... release to transcribe" |
| Transcribing | Spinner, button disabled | "Transcribing..." |
| Ready | Returns to idle; transcript inserted into input field | — |
| Permission denied | Button disabled, error banner shown | See Copywriting Contract |

### TTS Playback

| State | Visual |
|-------|--------|
| Generating | Spinner below Jarvis bubble: "Generating voice response..." |
| Playing | `st.audio(autoplay=True)` widget shown inline below Jarvis bubble |
| Complete | Audio widget remains visible for replay; no additional state change |
| Failed | Info banner below bubble (not error — graceful degradation): see Copywriting Contract |

### Text Input Field

- Placeholder: "Type a command or use the mic..."
- On STT transcript ready: field value is set to transcript text; user can edit before submitting
- On submit (Enter or Send button): field clears, message appended to chat history

## Implementation Guidance for Executor

### Extend `src/ui/styles.py`

Append new CSS classes (`.chat-container`, `.chat-bubble`, `.chat-bubble.coordinator`,
`.chat-bubble.jarvis`, `.chat-meta`, `.intent-badge`, `.mic-button-active`, `.voice-panel`)
to the `CUSTOM_CSS` string at the bottom of the existing `<style>` block.
Add `--intent-badge-color: #F59E0B;` to the `:root` block alongside existing semantic color vars.
Do not create a separate CSS file. Do not modify any existing class definitions.

### New file: `src/ui/command_center.py`

Follow the existing tab module pattern: export a single `render_command_center_tab()` function.
Import from `src.voice.stt` and `src.voice.tts` — NOT directly from kittentts/faster-whisper.
Use `st.markdown(..., unsafe_allow_html=True)` to render `.chat-bubble` HTML fragments.
Use `st.audio(wav_bytes, format="audio/wav", autoplay=True)` for TTS playback.

### Chat history rendering

Iterate `st.session_state["conversation_history"]` (list of dicts with keys `role`, `text`,
`timestamp`, `intent`). Render each entry as a `.chat-bubble.coordinator` or `.chat-bubble.jarvis`
HTML block via `st.markdown`. Do not use `st.chat_message()` — it conflicts with custom bubble
alignment and the intent badge pattern.

### Voice panel layout

Use `st.columns([6, 1])` — left column: text input + Send button; right column: mic button.
Wrap both in a `<div class="voice-panel">` injected before the columns.
