Add a `build.rollupOptions.output.manualChunks` function to `vite.config.ts`. The function receives `(id)` and returns a chunk name based on the module path. ALL existing config (plugins, resolve, server, assetsInclude) MUST remain unchanged.

Add this `build` key at the top level of the `defineConfig` object (after `assetsInclude`):

```typescript
build: {
  rollupOptions: {
    output: {
      manualChunks(id: string) {
        if (id.includes("node_modules")) {
          if (
            id.includes("react-dom") ||
            id.includes("react-router") ||
            (id.includes("/react/") && !id.includes("react-"))
          ) {
            return "vendor-react";
          }
          if (id.includes("recharts") || id.includes("d3-")) {
            return "vendor-charts";
          }
          if (id.includes("@mui/") || id.includes("@radix-ui/")) {
            return "vendor-ui";
          }
          if (id.includes("@emotion/")) {
            return "vendor-emotion";
          }
        }
      },
    },
  },
},
```

The 4 chunks are:
- `vendor-react`: react, react-dom, react-router (core runtime)
- `vendor-charts`: recharts and its d3 sub-dependencies
- `vendor-ui`: @mui/* and @radix-ui/* component libraries
- `vendor-emotion`: @emotion/* CSS-in-JS styling engine

This splits the ~1,042 kB monolithic vendor chunk into 4 smaller chunks, each expected to be under 500 kB.
